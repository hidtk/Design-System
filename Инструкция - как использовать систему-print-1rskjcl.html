Акметрон · Design System

Как подключить и использовать систему

Эта дизайн-система самодостаточна: один файл стилей styles.css, собранная библиотека компонентов и два готовых шаблона. Ниже — три способа применить её в других проектах: от самого простого до варианта для разработчиков.

Способ 1 · Рекомендуется

01Привязать как дизайн-систему

Самый простой путь — никакого кода. В новом проекте система подключается одним выбором, и дальше всё подтягивается автоматически.

  1. Создайте новый проект (или откройте существующий).
  2. В настройках проекта прикрепите дизайн-систему «Акметрон Design System» — так же, как она прикреплена к этому разговору. В списке найдёте её по названию или по ID b1016101…b534c39b.
  3. Просто пишите задачу обычным языком: «Сделай лендинг для направления ГНСС» или «Собери дек на 10 слайдов про метрологию».
  4. Готово. Цвета, шрифты, отступы, компоненты и шаблоны применяются автоматически — изобретать стиль заново не нужно.
Почему так лучше всего: привязка держит единый визуальный язык во всех проектах. Поменяете токен в системе — и обновление можно прокатить по всем потребителям.
Способ 2 · Для быстрого старта

02Начать с готового шаблона

Если нужен конкретный формат — дек или лендинг — стартуйте прямо с шаблона. В привязанном проекте система лежит в папке _ds/<имя>.

  1. Скопируйте папку templates/deck/ или templates/landing/ в свой проект.
  2. Откройте рядом лежащий ds-base.js и поправьте одну строку base, чтобы она указывала на привязанную систему:
// templates/landing/ds-base.js — было (внутри самой системы):
const base = '../..';

// стало (в проекте-потребителе, страница в корне):
const base = '_ds/akmetron';

// если страница лежит на уровень глубже:
const base = '../_ds/akmetron';

Дальше правьте контент в index.html — массив направлений, заголовки, метрики. Разметка и компоненты уже на месте.

Способ 3 · Для разработчиков

03Скачать или вынести в GitHub

Нужен, только если систему надо отдать наружу — команде разработки или на отдельный хостинг. Скачиваете архив проекта и заливаете в репозиторий.

# распакуйте архив, зайдите в папку:
git init
git add .
git commit -m "Акметрон Design System"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПО.git
git push -u origin main

В коде потребителя подключение — две строки в <head>:

<!-- глобальные стили: токены, шрифты, базовые правила -->
<link rel="stylesheet" href="styles.css">

<!-- библиотека компонентов (после React + ReactDOM UMD) -->
<script src="_ds_bundle.js"></script>
Компоненты React живут в глобале window.DesignSystem_176d4c. Для иконок дополнительно подключите Lucide: <script src="https://unpkg.com/lucide@latest">.
Шпаргалка

04Что под рукой

Ключевые токены

  • --blue · --blue-light — функциональный акцент «смотри сюда»
  • --text-1 → --text-2 → --text-3 — три ступени текста
  • --panel-top / --panel-bottom — градиент панелей и карточек
  • --alert-stripe, --alert-text — только для рисков и проблем
  • --font-display (Montserrat) · --font-body (Inter)

Компоненты — пример вызова

const { Button, Card, Stat, Callout, Icon } = window.DesignSystem_176d4c;

<Card>
  <Stat value="67 ГГц" label="диапазон" />
  <Button>Получить расчёт</Button>
</Card>
Главные правила бренда: цвет функционален, не декоративен · никаких эмодзи · заголовки без точки · числа конкретны (<1 ч, не «быстро») · ~60% воздуха в макете.
